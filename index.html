Tutorial: Creating Your Portfolio Website with GitHub Pages and Notion

Introduction:

It‚Äôs no secret that we are obsessed with Notion‚Äôs UI - and can make the most beautiful boards around.

But what if we could leverage this expertise to create our own digital portfolios for free?

This tutorial will guide you through just that. Combining the power of GitHub for hosting and Notion for content creation, you'll have a professional and easily maintainable portfolio to showcase your skills and accomplishments.

Why not use the Notion ‚ÄòPublish to Web‚Äô feature?

Notion does have a feature to host websites directly from the web app, and this is another option to consider. However, I prefer hosting my site on GitHub for the added benefit of customizability and seamless integration with my software projects on GitHub. Additionally, GitHub is open-source, meaning you will never have to worry about having to pay a premium to keep your site up.

Prerequisites:

  1. GitHub Account
  1. Notion Account
  1. Basic understanding of web development concepts (HTML, CSS)

Steps:

Step 1: Set Up Your GitHub Repository

  1. Create a new repository on GitHub, naming it my-portfolio. Make sure the repository is set to Public - this will ensure your site can be viewed by anyone on the web.
  1. Optional: Clone the repository to your local machine using Git. Alternatively, it is just as easy to use the web for this tutorial, and I recommend doing this if you do not already have Git downloaded on your local machine.

Step 2: Design Your Portfolio in Notion

  1. Create a new Notion page for your portfolio.
  1. Design each section and subsection of your portfolio using Notion's interface. Feel free to use all of Notion‚Äôs available UI features in this step - they will transfer seamlessly to your digital portfolio on the web.
  1. Don‚Äôt forget include details about your projects, skills, education, Resume/CV, and any other relevant information.

Step 3: Export Notion to HTML

  1. Open your Notion page.
  1. Click on the "Share" button in the top-right corner.
  1. Select "Export" and choose "HTML" for the Export Format option. You can select what content you want to transfer to your site (‚ÄôEverything‚Äô is a safe choice) and what default views you would like your databases to have. Make sure you have toggled Include Subpages on. This is the key to multiple subsections and hyperlinks for your portfolio. Optionally, you can toggle Create folders for subpages on, but this is optional.
  1. Save the exported ZIP file to your local machine. Extract the contents of the file to a folder on your local machine.

Step 4: Push Changes to GitHub

  1. Upload your HTML File:
    • Open your GitHub repository on the web.
    • Navigate to the "Code" tab.
    • Click on the "Add file" dropdown and select "Upload files."
    • Drag and drop the contents of the ZIP file you just extracted into the upload area or choose the files manually. Make sure you upload everything, so your site is complete.
    • Once uploaded, you'll see the file listed in the repository, along with another folder containing the subpages and other content.
    • Rename the HTML file in the main directly to ‚Äúindex.html‚Äù, so the site knows where to start (this is called the root page and it‚Äôs essential for your site to run properly).
  1. Commit Changes:
    • Scroll down, and you'll find the "Commit changes" section. Ensure the "Commit directly to the main branch" option is selected.

Step 5: Enable GitHub Pages

  1. Go to your GitHub repository settings.
  1. Scroll down to the "GitHub Pages" section.
  1. Choose the "main" branch as the source.

Now, your portfolio website is live at https://<your-username>.github.io/my-portfolio. Any updates you make to your Notion page or GitHub repository will automatically reflect on your live portfolio.
